# sample35-capacitor-artifact-import

`sample35` is a Capacitor-ready React sample that proves an external app can consume Mtool artifacts directly.

It is not generated by Mtool, and Mtool does not initialize Capacitor here. The sample app imports checked-in Mtool-style JSON fixtures and covers the Mtool-intended operation surface that a wrapper app is expected to understand.

## Scope

Covered Mtool operations:

- artifact import and artifact index review;
- optional `external-output.json` import and boundary review;
- project/app identity display;
- list, detail, and form screen rendering;
- field rendering with readonly/editable and required markers;
- local form draft state;
- required-field validation;
- `no-code-runtime-action-intent-v0` draft creation;
- mock/disabled submit handoff boundary;
- blocked/error state display;
- ownership boundary display.

Non-goals:

- generating a production React app;
- running `npx cap init`;
- creating `ios/` or `android/`;
- native build, signing, store submission, or device QA;
- covering every Capacitor plugin or permission feature.

## Mandatory validation

This gate does not install dependencies.

```bash
node sample/tutorials/sample35-capacitor-artifact-import/scripts/validate-sample.mjs
```

The validation checks the fixture schemas, source imports, operation coverage markers, and that `ios/` / `android/` are not present.

It also checks that the optional `external-output.sample.json` fixture keeps `mtool_no_code` as the supported baseline, marks `external_no_code` as additive, and does not allow Capacitor initialization, dependency installation, or native project generation without an explicit external-owner step.

## Optional external-owner checks

These are intentionally optional because they require package installation and are owned by the consuming app/project, not by Mtool itself.

```bash
cd sample/tutorials/sample35-capacitor-artifact-import
npm install
npm run build
npm run cap:sync
```

`cap:sync` is included to show the expected external-owner route. It should only be used after the external app owner chooses app IDs, native settings, signing policy, and environment configuration.
