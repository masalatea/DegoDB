# Sample35 external-output consumer evidence

## Status

`EF_M4_DONE`

## Purpose

Complete EF-M4: prove that the existing Capacitor-ready sample can consume the optional `external-output.json` packet as an input artifact without making Mtool initialize or mutate an external app project.

## What changed

Updated `sample/tutorials/sample35-capacitor-artifact-import/`:

- added `src/mtool-artifacts/external-output.sample.json`;
- imported the fixture through `src/mtoolArtifacts.ts`;
- displayed the optional external-output boundary in `MtoolArtifactSummary`;
- added `external_optional_output` to the sample bundle manifest fixture;
- extended `scripts/validate-sample.mjs` to validate the optional external-output contract;
- updated the sample README.

## Boundary proven

The sample now verifies that:

- `external-output.sample.json` uses `mobile-external-optional-output-v1`;
- the output mode is `external_no_code`;
- the target is `react_web_capacitor`;
- `mtool_no_code` remains the supported baseline;
- no replacement claim is made;
- source artifact refs still point back to the no-code runtime / bridge fixture;
- server authority remains with Mtool/server-side gates;
- React shell / routing / component / Capacitor/native preparation remain external-owner responsibilities;
- Capacitor initialization and dependency installation require explicit external-owner action;
- `ios/` and `android/` are still absent from the checked-in sample.

## Not claimed

This slice does not:

- run `npm install`;
- run `npm run build`;
- run `npx cap sync`;
- create a native iOS or Android project;
- choose plugins, signing, store submission, or production frontend architecture.

Those remain external-owner gates.

## Current conclusion

EF-M4 is complete.

The optional external no-code/tool packet is now:

1. generated by Mtool tooling;
2. documented as durable external output;
3. discoverable through the bundle manifest;
4. consumable by the Capacitor-ready sample as a static input artifact.

Next recommended step is a commit checkpoint before expanding the external-output lane further.
