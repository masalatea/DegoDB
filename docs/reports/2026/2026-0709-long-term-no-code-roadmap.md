# Long-Term No-Code Roadmap

Date: 2026-07-09

Status: `ROADMAP`

## Summary

The long-term no-code direction is to make the sample UIs no-code first, then dogfood Mtool itself, and finally use AI to normalize structure, analyze relationships, and generate usable no-code UI directly from materials.

## Phases

| Phase | Direction | Outcome |
| --- | --- | --- |
| L1 | Sample UI no-code conversion | Practical no-code gaps become visible against real samples, and the sample set becomes useful no-code proof. |
| L2 | Mtool self no-code | Mtool's own UI flows become no-code driven, creating a stronger dogfooding loop. |
| L3 | AI structural normalization | AI turns messy user materials into normalized structures, then progresses toward relationship and ontology-like analysis. |
| L4 | Instant no-code generation from materials | Users can provide materials with little explanation; AI analyzes structure, answers comprehensively, and produces a usable no-code UI. |

## Product Intent

- Help users who are not familiar with IT choose this tool and still get structured views of their material.
- Let AI start analysis automatically when appropriate.
- Make the generated structure inspectable through no-code UI.
- Let users ask the AI directly and receive comprehensive answers grounded in the normalized structure.

## Boundary

- This roadmap does not replace the immediate #537 availability preflight.
- AI structural normalization is a later phase after sample UI no-code and Mtool self no-code dogfooding.
- No AI-driven automatic mutation or generated UI execution is enabled by this roadmap record.
