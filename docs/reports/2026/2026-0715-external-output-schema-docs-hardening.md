# External output schema/docs hardening

## Status

`EF_M3_DONE`

## Purpose

Complete EF-M3: make the optional `external-output` packet discoverable and durable for users, external tools, and AI/code-builder workflows.

## What changed

Added permanent documentation:

- `docs/external-no-code-output.md`

Updated permanent indexes/docs:

- `docs/README.md`;
- `docs/mobile-react-wrapper-target-contract.md`.

Updated bundle manifest behavior:

- `external-output` is now included in the mobile wrapper bundle manifest as `external_optional_output`.

This makes the optional external output visible from the bundle without making it mandatory and without replacing `mtool_no_code`.

## Bundle decision

Decision: include `external-output` in the bundle manifest.

Reason:

- `external-output` is the clearest machine-readable entry point for external no-code/tool consumers;
- omitting it from the bundle would make the new optional output easy to miss;
- including it does not imply that external output is mandatory;
- the manifest remains an index/checklist only and still creates no external app project.

## Boundary preserved

Still not generated by Mtool:

- production React app;
- `package.json`;
- `capacitor.config.ts`;
- `ios/`;
- `android/`;
- dependency installation;
- native build;
- signing;
- store submission.

`mtool_no_code` remains the supported baseline.
